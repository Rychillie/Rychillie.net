Tenho paixão por desenvolver 'Portfólios' e explorar inovações em sites pessoais. Acredito que criar uma plataforma personalizada é a maneira mais eficaz de apresentar habilidades. Esta não é minha primeira incursão na criação de uma presença online pessoal. Honestamente, não quero que meu site pessoal se torne apenas mais uma entidade perdida na vastidão da internet. Quero que seja uma representação autêntica de quem sou e das minhas habilidades. É por isso que escolhi desenvolver esta plataforma complexa, que, enquanto escrevo este artigo, assume a forma de um blog multilíngue com um Sistema de Gerenciamento de Conteúdo (CMS) completamente personalizado.

## Desenvolvimento Fullstack

Em termos de Fullstack, começarei explicando as ferramentas que uso para tornar meu projeto abrangente, cobrindo tanto o BackEnd quanto o FrontEnd. Para a construção do BackEnd e FrontEnd, minha escolha principal é NextJS, na versão 14. NextJS é essencialmente um Framework ReactJS que se destaca em Renderização no Lado do Servidor. Em outras palavras, permite a renderização de conteúdo no servidor. A presença incorporada de NodeJS no NextJS permite seu uso como uma solução Fullstack.

Para simplificar ainda mais o desenvolvimento, recorro ao [Prisma ORM](https://www.prisma.io/). Este framework desempenha um papel crucial no mapeamento, tipificação e solicitação de dados do meu banco de dados, que é essencialmente [Planetscale](https://planetscale.com/). Essa combinação de tecnologias não apenas proporciona uma experiência de desenvolvimento prática, mas também estabelece uma base robusta para a implementação eficiente de recursos em meu projeto.

## Explorando NextJS 14 e Server-Components

Ao adotar a versão 14 do NextJS, abro as portas para usar Server-Components, um recurso inovador que proporciona a capacidade de fazer solicitações diretamente no servidor sem expor informações sensíveis ao usuário. Esses componentes não apenas fornecem uma camada adicional de segurança, mas também apresentam uma abordagem mais eficiente para a entrega de conteúdo estático, resultando em uma experiência mais rápida e responsiva para os usuários.

## Prisma e Planetscale

Incorporando o Prisma como meu Mapeamento Objeto-Relacional (ORM) e Planetscale como meu banco de dados principal, estabeleço uma infraestrutura sólida para o gerenciamento eficiente e seguro de dados em meu projeto. Prisma simplifica a interação com o banco de dados, fornecendo uma poderosa camada de abstração para mapeamento de dados e consultas tipificadas, melhorando a produtividade e a clareza do código.

Vale ressaltar que o Planetscale, além de oferecer escalabilidade e desempenho excepcionais, conta com um plano gratuito notável que merece atenção especial. Essa opção gratuita não apenas possibilita projetos iniciais, mas também se destaca como uma solução robusta para iniciativas em crescimento. Sua infraestrutura distribuída lida de maneira eficiente com volumes consideráveis de dados, garantindo uma experiência confiável para os usuários finais. Ao considerar o plano gratuito do Planetscale, não apenas otimizo custos, mas também asseguro uma base sólida para a expansão futura do meu projeto.

## Explorando FullStack com Next.js Server-Actions

Vamos agora nos aprofundar no funcionamento prático das Server-Actions no Next.js, incorporando Prisma para fazer solicitações ao banco de dados. O exemplo atualizado destaca a simplicidade e eficácia desta abordagem, conforme demonstrado no seguinte trecho de código:

```tsx title="@/lib/actions/articles.tsx" caption="Example of Server-Action with Prisma"
'use server';

import prisma from '@/lib/prisma';

// Começando com a criação de uma função assíncrona
export async function getPost({ slug }) {
  // Nós recebemos apenas o SLUG da postagem como parâmetro
  return await prisma.posts.findMany({ where: { slug } });
  // Nós usamos o SLUG para fazer uma consulta ao banco de dados com Prisma
}
```

Neste exemplo, ao incorporar o marcador _**"use server"**_, indicamos explicitamente que se trata de uma Server-Action. A função _**getPost**_ usa Prisma para fazer uma consulta ao banco de dados, filtrando posts com base no _**slug**_ fornecido. Essa implementação exemplifica a elegância da integração entre Server-Actions e Prisma no Next.js, proporcionando uma maneira eficiente e direta de lidar com operações do lado do servidor.

## CMS Personalizado com TIPTAP

Minha jornada no desenvolvimento desta plataforma também inclui a criação de um Sistema de Gerenciamento de Conteúdo (CMS) totalmente personalizado, comparável a um "WordPress pessoal". A força motriz por trás dessa experiência única está no poderoso editor de texto, TIPTAP.

Ao adotar o [TIPTAP](https://tiptap.dev/), uma biblioteca para construção de editores WYSIWYG, aprimorei significativamente a experiência de edição e criação de conteúdo. Personalizar a entrega de conteúdo com Next.js, Prisma e outras tecnologias não apenas simplificou a administração do meu conteúdo, mas também ofereceu uma interface intuitiva. Isso me permite criar e editar artigos facilmente, a qualquer hora e em qualquer lugar.

Com a flexibilidade do TIPTAP, explorei recursos avançados de formatação de texto, proporcionando aos usuários uma experiência de leitura única e amigável. Essa iniciativa reforça a ideia de que, combinando tecnologias poderosas, podemos criar soluções personalizadas que atendem às necessidades específicas do projeto.

## Criando Elementos com Tailwind CSS, CVA e Radix UI

Além do CMS personalizado, a personalização do design do meu site é uma peça essencial do que construí. Usando a abordagem de primeiro utilitário do Tailwind CSS, estilizei cada componente de forma eficiente, mantendo o código limpo e organizado.

A integração com a [**Class Variance Authority** (ou CVA)](https://cva.style/) para Tailwind facilitou ainda mais a gestão da lógica de estado e estilo, proporcionando uma experiência de desenvolvimento mais suave, permitindo assim a criação de um mini-sistema de design. Além disso, a incorporação do [Radix UI](https://www.radix-ui.com/) ofereceu uma coleção de componentes acessíveis e prontos para uso, enriquecendo a interface do meu site com elementos interativos e responsivos.

Essa sinergia entre Tailwind CSS, CVA e Radix UI não apenas simplifica o processo de personalização, mas também eleva o design do meu site para um novo nível, refletindo minha visão estética de uma maneira única e marcante.
